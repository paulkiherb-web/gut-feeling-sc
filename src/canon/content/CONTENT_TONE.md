# Content Tone Governance
## Personal State OS — Voice, Register, and Language

> The tone of this product is its most important invisible feature.
> A precisely written product feels trustworthy and intelligent.
> A carelessly written product feels generic and amateur.
> There is no AI-generated copy in this product.
> There is no wellness marketing language in this product.

---

## 1. Core Tone Definition

The voice of this product is:

| Quality     | Description                                                          |
|-------------|----------------------------------------------------------------------|
| Composed    | Never rushed, never breathless, never reactive                       |
| Concise     | Every sentence contains exactly what is needed and nothing more      |
| Intelligent | Assumes the user is an adult who can handle precise information      |
| Calm        | Never uses alarm or urgency language for manageable situations       |
| Precise     | Uses specific language rather than vague affirmations                |

**The product speaks like a well-informed, calm advisor —
not like a wellness coach, not like an AI assistant.**

---

## 2. Tone Register by State

The adaptive layer shifts verbosity but **never** changes the fundamental tone.
All registers remain: composed, precise, reassuring, intelligent.

| Register  | When Used                          | Verbosity | Rationale | Impact |
|-----------|------------------------------------|-----------|-----------|--------|
| terse     | depleted, overloaded               | Minimal   | Hidden    | Hidden |
| concise   | fragile, recovering                | Moderate  | Brief     | Brief  |
| standard  | stable, focused, optimized         | Full      | Shown     | Shown  |

**What stays the same across all registers:**
- The language is never alarming
- The language is never congratulatory
- The language is never fake-empathetic
- The language is never motivational

## 3. Permitted Language Patterns

### 3.1 — For positive or improving states

```
✓ "Recovery improving over the last 3 days."
✓ "Readiness is high — full context available."
✓ "Sleep quality supports today's targets."
✓ "Trajectory: improving."
```

### 3.2 — For neutral or stable states

```
✓ "Readiness: 72. Within your typical range."
✓ "Sleep: adequate. No adjustments indicated."
✓ "No significant deviations from baseline."
```

### 3.3 — For negative or declining states

```
✓ "Recovery lower than average. Reduced activity recommended."
✓ "Sleep quality below threshold. Prioritize rest today."
✓ "Readiness is lower than your typical range."
✓ "One recommendation for today."
```

### 3.4 — For risk and prediction

```
✓ "Risk: elevated if current pattern continues."
✓ "At current trajectory, recovery may decline by Thursday."
✓ "Consider reducing training load this week."
```

---

## 4. Forbidden Language Patterns

### 4.1 — Cringe wellness language

```
✗ "You're doing amazing!"
✗ "Keep up the great work!"
✗ "Your body is your temple."
✗ "Listen to your body."
✗ "You deserve to feel good."
✗ "Self-care is not selfish."
✗ "Honor your health journey."
```

### 4.2 — Motivational clichés

```
✗ "Don't give up!"
✗ "Every step counts!"
✗ "You're crushing it!"
✗ "Push through!"
✗ "Believe in yourself!"
✗ "Small habits, big results!"
```

### 4.3 — Hype AI language

```
✗ "AI-powered insights"
✗ "Smart recommendations powered by machine learning"
✗ "Our intelligent algorithm has detected..."
✗ "Your personal AI coach"
✗ "Backed by advanced AI"
✗ "Deep analysis of your data"
```

### 4.4 — Emoji in UX

```
✗ 🎉 for any achievement
✗ 💪 for readiness or performance
✗ 🌟 for any recommendation
✗ ❤️ for any health metric
✗ Any emoji in a system message or recommendation
```

**Exception:** Emoji are permitted in user-generated content contexts
(e.g., if a user's note or custom entry contains emoji).

### 4.5 — Fake empathy

```
✗ "We understand this is hard."
✗ "We know how challenging this can be."
✗ "You're not alone in this."
✗ "We're here for you."
✗ "It's okay to have off days."
```

### 4.6 — Alarm language

```
✗ "CRITICAL: Your recovery is low!"
✗ "WARNING: You must rest today."
✗ "ALERT: Risk level is high."
✗ "DANGER: Sleep deficit detected."
```

**Note:** For genuinely critical health situations, escalate to
clinical language: "Consult a healthcare provider."
Never use alarm language for routine physiological variation.

## 5. CTA Copy Rules

| CTA Type    | Formula               | Examples                              |
|-------------|-----------------------|---------------------------------------|
| Primary     | Verb + object         | "Scan now", "View details", "Add to plan" |
| Secondary   | Verb + context        | "See full history", "View alternatives" |
| Tertiary    | Noun or verb phrase   | "Learn more", "See all", "Dismiss"    |

**Forbidden in CTA copy:**
- Exclamation marks
- "Click here"
- "Get started" (too generic)
- "Amazing", "Great", "Incredible"
- Questions ("Want to see more?")

---

## 6. Number and Metric Formatting

- Readiness scores: integer only (e.g., "72", not "72.3")
- Recovery scores: integer only
- Sleep duration: "7h 20m" format
- Percentages: integer with "%" (e.g., "85%")
- Dates: relative when possible ("Today", "Yesterday", "3 days ago")
- Avoid "approx." or "~" — be precise or omit

---

## 7. Copy Length Limits

| Element            | Character Limit | Word Limit |
|--------------------|-----------------|------------|
| Card title         | 50 chars        | 7 words    |
| Card body (1 line) | 80 chars        | 14 words   |
| Card body (2 lines)| 160 chars       | —          |
| Recommendation     | 120 chars       | 20 words   |
| Insight            | 200 chars       | —          |
| CTA label          | 24 chars        | 3 words (primary) |
| Section header     | 32 chars        | 4 words    |
| Navigation label   | 12 chars        | 1 word     |

---

## 8. Localization Notes

- Russian copy must maintain the same composed, calm register
- Do not use diminutives in Russian that feel informal or patronizing
- Avoid machine-translation feel — all Russian copy must be reviewed by a native speaker
- Technical terms may be kept in English when Russian equivalents are awkward
